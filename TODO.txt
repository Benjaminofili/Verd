Hybrid AI Scanning Architecture
Our goal is to build a robust crop scanning feature that works deep in the African farmlands (where internet is spotty or non-existent) but leverages the massive power of Google's Gemini models when a connection is available.

Core Architectural Approach: The "Hybrid Router"
We will implement a clean AIService acts as a router. When the user takes a photo of a crop, this service will check the device's internet connection status and route the image analysis to one of two engines.

1. The Online Path (Firebase + Gemini)
When connectivity_plus detects an active internet connection:

The app uploads the crop image to Firebase Storage (/scans/{uid}/{scanId}.jpg).
The upload triggers the Multimodal Tasks with the Gemini API Firebase Extension (which we will configure).
Gemini analyzes the image for crop type and disease, writing the structured JSON result to a Firestore document (/users/{uid}/scans/{scanId}).
The app listens to this Firestore document in real-time and updates the UI instantly.
2. The Offline Path (Local TFLite Model)
When connectivity_plus detects no connection (or poor ping):

The app skips the Firebase upload entirely.
The image is passed to a locally bundled TensorFlow Lite (.tflite) model.
The model runs inference directly on the phone's CPU/NPU.
The result is saved to the local Hive database (for syncing later) and displayed immediately.
